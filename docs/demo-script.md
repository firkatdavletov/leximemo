# Demo Script

## Цель демо

За 3-5 минут показать, что LexiMemo решает практическую задачу изучения слов, уже имеет завершенный MVP-цикл и готов к использованию как веб-приложение и installable PWA.

## Сценарий показа

1. Главная страница
- Коротко сказать, что это дипломный MVP для изучения слов по карточкам.
- Упомянуть, что приложение можно установить как PWA.

2. Логин
- Войти под демо-пользователем.
- Подчеркнуть, что у каждого пользователя свои колоды и свои данные.

3. Список колод
- Показать несколько подготовленных колод.
- Обратить внимание на блок статистики, streak и achievements.

4. Открыть колоду
- Показать список карточек.
- Показать язык озвучки, интервалы и даты следующего повторения.
- Нажать browser TTS на одной карточке.

5. AI-генерация
- Открыть AI-блок.
- Показать prompt и preview карточек.
- Объяснить, что пользователь сначала видит preview и только потом сохраняет карточки.

6. Учебная сессия
- Нажать `Начать обучение`.
- Открыть ответ по карточке.
- Поставить одну из оценок.
- Показать, что сессия двигается вперед по due-карточкам.

7. Завершение сессии
- Показать экран завершения.
- Отметить, что сервер обновляет progress, review history, streak и achievements.

8. PWA
- Если показываете на desktop, открыть install prompt или значок установки в браузере.
- Коротко сказать, что проект installable, но без тяжелой offline-синхронизации.

## Что важно проговорить комиссии

- Приложение закрывает полный пользовательский сценарий: от регистрации до review-сессии.
- Бизнес-логика повторения и прогресса хранится на сервере, а не только в клиенте.
- Проект выбран как единый Next.js fullstack, чтобы сократить инфраструктурную сложность для MVP и диплома.
- PWA повышает практическую ценность: приложение можно установить и использовать как более "нативный" инструмент.

## Практическая ценность

- Помогает учить слова небольшими порциями.
- Поддерживает регулярность за счет streak и повторений по расписанию.
- Позволяет быстро создавать контент вручную или через AI.
- Подходит как база для дальнейшего развития в полноценный EdTech-продукт.

## Что честно озвучить как ограничения

- Нет полноценного offline-first режима.
- Нет push-уведомлений.
- TTS зависит от браузера и устройства.
- AI-генерация зависит от внешнего сервиса.
- Алгоритм интервального повторения пока упрощенный, но уже рабочий для MVP.
